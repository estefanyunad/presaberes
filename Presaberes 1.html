Formulación del problema

¿Cómo organizar el inventario de Texas Electronics, dado a conocer listados de compras, precio de venta al público e impuestos a través del diseño de un aplicativo web ?

Objetivo General

Diseñar un aplicativo web que genere listados de compras, inventarios actualizados, precio al público incluyendo para este la tasa de impuestos de la empresa Texas Electronics.

Objetivos Específicos

1. Generar listados de compras de productos agotados y próximos a agotarse.

2. Mostrar inventarios actualizados según la existencia en bodega.

3.Diseñar página exclusiva para la venta directa de los productos que ofrece Texas Electronics.

Metodología

1- Investigación

Se debe elaborar un análisis y una evaluación general del manejo de todo el almacén de Texas Electronics para así conectar el personal encargado de las áreas en las que principalmente se están evidenciado las falencias con el fin de mantener listados de agotados, inventarios y la ejecución de la página de venta directa.

2- Planificación

En este paso se creará los prototipos del aplicativo web teniendo en cuanta las necesidades evidenciadas en el análisis de la empresa Texas Electronics dándole prioridad a la creación de listados de compras para los agotados o próximos a agotarse, inventario y la página de venta directa con los precios teniendo en cuenta los impuestos.

3- Arquitectura del sitio y contenido

Para este paso tenemos que concluir con las necesidades principales que se deben evidenciar en el aplicativo Web. Debemos crear la estructura del aplicativo, el diseño y navegabilidad del mismo.

4- Hacer pruebas

En este paso es donde ya teniendo la estructura con navegabilidad se comienzan hacer la pruebas del aplicativo evidenciando las posibles falencias, dándole así solución y mejorando los requerimientos específicos exigidos por la empresa Texas Electronics.

5 - Operar, mantener y evaluar

En este paso es en donde se pone en funcionamiento el aplicativo web en la empresa, se hacen capacitaciones al personal que quedará encargado de alimentar el sitio y se deberá evaluar por el personal la manejabilidad de este y su estructura. Finalidad del proyecto Con este aplicativo Web buscamos dar solución a los inconvenientes que se venían presentando en el inventario, compras y pedidos de clientes generando pérdidas significativas a largo plazo para la empresa Texas Electronics.

The client server architecture

The main elements of the client server architecture are precisely the element called client and the other element called server. "For example in a multimedia environment, the client element would be the device that can watch the video, pictures and text, or reproduce the audio distributed by the server element. On the other hand, the client can also be a personal computer or a smart television that has the ability to understand digital data. In this case the server element is the depository of digital video, audio, digital photographs and text and distributes them on demand to be a machine that has the ability to store the data and execute all the software that it provides to the client.

The MVC or Model-View-Controller is a software architecture pattern that, using 3 components (Views, Models and Controllers) separates the application logic from the view logic in an application. It is an important architecture since it is used in both basic graphic components and enterprise systems; most modern frameworks use MVC (or some adaptation of the MVC) for architecture, among them we can mention Ruby on Rails, Django, AngularJS and many others.
